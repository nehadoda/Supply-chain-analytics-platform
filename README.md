# Analytics Engineering — Ingestion to Insights

> End-to-end analytics engineering project — Python · dbt · DuckDB · Power BI

---

## What I Built

A complete analytics engineering pipeline that ingests data from three real-world sources, transforms and models it using dbt, and delivers business-ready tables to Power BI.

> **Data note:** Microsoft sample dataset · Sept 2013 – Dec 2014 · 6 products · 5 segments · 5 countries

## Key Technical Decisions

- **Staging layer** separates raw cleaning from business logic — no transformations mixed together
- **dbt refs** used throughout — dbt manages full dependency and execution order automatically
- **NULLIF** used in profit margin calculation to prevent division by zero
- **CROSS JOIN** used in price comparison — competitor products are not directly mapped to internal products
- **DuckDB** chosen for zero-infrastructure local analytical storage
- **CSV export** used for Power BI import mode — avoids DuckDB single-writer file locking
- **Year-on-year comparison avoided** — 2013 has only 4 months of data, making it misleading
- **Single refresh script** — one command rebuilds the entire pipeline end to end

## Data Models

| Model | Description | Key Output |
|-------|-------------|-----------|
| `stg_sales` | Cleaned sales data | Typed columns, parsed dates |
| `stg_currency_rates` | Flattened API JSON | Validated exchange rates |
| `stg_competitor_prices` | Cleaned scraped data | Stripped symbols, cast prices |
| `dim_product` | Unique products | Filter table |
| `dim_country` | Unique countries | Filter table |
| `dim_currency` | Unique currencies | Filter table |
| `fct_sales_with_currency` | Sales + currency rates | Multi-currency revenue |
| `fct_profit_analysis` | Aggregated profitability | Profit margin % |
| `fct_price_comparison` | Us vs competitor | Price position |
| `fct_currency_impact` | Local currency revenue | Currency impact in USD |
